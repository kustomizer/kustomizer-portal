# Documentation (Baseline)

## When to document

- New public API (function, module, endpoint)
- Non-obvious behavior, constraints, or trade-offs
- Architectural decisions that impact multiple packages

## Preferred locations

- Behavior/spec: `openspec/specs/`
- Change intent/tasks: `openspec/changes/<change>/`
- Developer workflow: `docs/ai/` (preferred) or repo root docs
